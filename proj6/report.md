**UNIX I/O and Process Communication**

**Overview**

The goal of this project was to implement a multi-process system using UNIX I/O system calls in C. The program creates five child processes that communicate with a parent process through pipes. The parent process uses the `select()` system call to monitor multiple pipes simultaneously and write received messages to an output file.

The system runs for 30 seconds. During this time, the child processes generate messages that are collected by the parent and written to `output.txt` with timestamps.

---

**System Design**

The program consists of one parent process and five child processes.

The parent process creates five pipes and forks five children. Each child process is connected to the parent through a dedicated pipe. The parent closes all write ends of the pipes and listens for incoming messages using `select()`.

The first four children act as message generators. Each generator child repeatedly sleeps for a random amount of time between 0 and 2 seconds. After waking up, the child generates a timestamped message and writes it to the parent through its pipe. This continues until 30 seconds have elapsed.

The fifth child process is interactive. It waits for input from the terminal using `select()` on `STDIN_FILENO` with a timeout. This approach prevents the program from blocking indefinitely while waiting for user input. When the user enters a line of text, the child process attaches a timestamp and sends the message to the parent through the pipe.

---

**Parent Process Behavior**

The parent process continuously monitors the read ends of the five pipes using `select()`. Whenever a pipe becomes readable, the parent reads the incoming data and prepends its own timestamp before writing the message to `output.txt`.

Each line in the output file therefore contains two timestamps:

1. The timestamp generated by the child process
2. The timestamp added by the parent process when the message is received

This allows us to observe the slight delay between message creation and message processing.

When a child process terminates and closes its pipe, the parent detects the end-of-file condition (`read()` returning 0). The parent then removes that pipe from the monitored set. The program terminates once all five child processes have exited.

---

**Implementation Details**

Several key UNIX system calls were used in this project:

* `fork()` to create child processes
* `pipe()` for inter-process communication
* `read()` and `write()` for pipe I/O
* `select()` for I/O multiplexing
* `gettimeofday()` for high-resolution timestamps

The timestamp format used in the program is `M:SS.mmm`, representing minutes, seconds, and milliseconds since the program started.

The interactive child uses `select()` with a timeout to ensure the program remains responsive and terminates correctly after 30 seconds even if no user input is provided.

---

**Issues Encountered**

One challenge in this project was handling multiple input sources without blocking the program. Initially, using `fgets()` directly caused the program to block indefinitely while waiting for user input. This was resolved by using `select()` with a timeout before calling `fgets()`.

Another issue involved handling partial or multiple messages read from pipes. Since a single `read()` call may contain multiple lines, the buffer is split using `strtok_r()` so each message can be processed individually.

Additionally, care was taken to flush the output file after each write to ensure that messages appear immediately in `output.txt`.

---

**Conclusion**

This project demonstrates how UNIX system calls can be used to implement a concurrent multi-process system with asynchronous communication. By combining pipes and the `select()` system call, the parent process is able to efficiently monitor multiple child processes and handle incoming messages in real time.
